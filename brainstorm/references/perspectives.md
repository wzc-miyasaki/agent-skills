# 头脑风暴思考角度清单

本文档提供多维度的思考角度，用于全面分析和澄清用户的想法。

## 1. 目标与价值维度

### 核心目标
- 想要解决什么核心问题？
- 为谁解决问题？（目标用户/受众）
- 成功的标准是什么？
- 预期产生什么价值？

### 业务价值
- 对业务的影响（收入、成本、效率）
- ROI 预期
- 商业模式考虑
- 市场定位

## 2. 用户与产品维度

### 用户体验
- 用户场景和使用流程
- 用户痛点和需求
- 易用性要求
- 可访问性考虑

### 产品设计
- 核心功能优先级
- MVP vs 完整方案
- 功能边界和范围
- 用户旅程设计

## 3. 技术实现维度

### 系统架构
- 整体架构设计（单体/微服务/Serverless）
- 技术栈选择
- 模块划分和职责
- 服务间通信方式

### 前端开发
- UI 框架选择（React/Vue/Angular）
- 状态管理方案
- 组件设计和复用
- 前端路由和导航
- 响应式设计需求

### 后端开发
- 编程语言和框架
- API 设计（RESTful/GraphQL/gRPC）
- 业务逻辑分层
- 中间件和工具链

### 数据层
- 数据模型设计
- 数据库选型（关系型/NoSQL/缓存）
- 数据一致性策略
- 数据迁移和版本管理

## 4. 质量与性能维度

### 性能优化
- 响应时间要求
- 并发量和吞吐量
- 缓存策略
- 性能监控方案

### 可靠性
- 可用性要求（SLA）
- 容错和降级策略
- 灾难恢复
- 监控和告警

### 测试策略
- 测试类型（单元/集成/E2E）
- 测试覆盖率目标
- 测试自动化方案
- 性能测试和压测

## 5. 安全与合规维度

### 安全考虑
- 认证和授权机制
- 数据加密（传输/存储）
- 常见安全威胁防护（XSS/CSRF/SQL注入）
- 敏感信息处理
- API 安全

### 合规性
- 数据隐私法规（GDPR/个人信息保护法）
- 行业标准和规范
- 审计日志要求
- 第三方服务合规

## 6. 运营与维护维度

### 可维护性
- 代码质量标准
- 文档完整性
- 日志记录策略
- 故障排查便利性

### 可扩展性
- 水平/垂直扩展能力
- 模块化和插件化
- 未来功能扩展预留

### DevOps
- CI/CD 流程
- 部署策略（蓝绿/金丝雀/滚动）
- 环境管理
- 配置管理

### 运维监控
- 系统监控指标
- 日志聚合和分析
- 告警机制
- 问题追踪流程

## 7. 资源与约束维度

### 时间约束
- 项目周期
- 里程碑和交付节点
- 快速迭代 vs 完美主义

### 人力资源
- 团队规模和技能
- 角色分工
- 知识传承

### 预算约束
- 开发成本
- 基础设施成本
- 第三方服务费用
- 运维成本

### 技术约束
- 现有技术栈限制
- 遗留系统兼容性
- 第三方依赖
- 技术债务

## 8. 风险与挑战维度

### 技术风险
- 技术可行性
- 技术方案不确定性
- 依赖第三方风险
- 技术过时风险

### 业务风险
- 市场变化
- 竞争对手
- 用户接受度
- 政策法规变化

### 项目风险
- 需求变更风险
- 资源不足风险
- 沟通协作风险
- 进度延期风险

## 9. 特定场景维度

### 日常对话/非技术场景
- 背景和上下文
- 相关利益方
- 现状和期望状态的差距
- 可能的替代方案
- 外部依赖和影响因素

### 方案设计
- 问题定义的准确性
- 多种方案比较
- 优缺点分析
- 决策依据和标准
- 实施路径

## 使用指南

### 如何选择角度
1. **初始评估**：快速识别主题所属领域（技术/产品/业务/日常）
2. **优先级排序**：根据用户关注点选择 3-5 个最相关的维度
3. **逐步深入**：从高层次问题开始，根据回答深入细节
4. **灵活调整**：根据对话进展动态调整关注角度

### 提问技巧
- 从开放式问题开始，逐步具体化
- 每次聚焦 1-2 个关键问题，避免信息过载
- 用假设性问题探索边界（"如果...会怎样？"）
- 用对比性问题澄清优先级（"A 和 B 哪个更重要？"）
- 识别隐含假设并提出质疑
